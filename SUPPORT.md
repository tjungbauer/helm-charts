# Support

## How to get help

- **Bug reports, questions, and ideas:** use **[GitHub Issues](https://github.com/tjungbauer/helm-charts/issues)** in this repository.
- **Contributions:** pull requests are welcome; see the root **README.md** for linting and layout notes.

## What to expect

These charts are published **for public use** under the project license, but support is **community-style, best effort** (I do this here for fun and re-usability):

- There is **no commercial SLA**, no guaranteed response time, and no obligation to implement feature requests.
- Help is provided **when maintainers and contributors have time**, not as a productized service.
- The **maintainer block** in each chart’s **Chart.yaml** (name, email, Artifact Hub metadata) identifies authorship and satisfies ecosystem conventions; it is **not** an offer of paid or vendor-grade support.

## Product and vendor support

Charts here automate or configure **Red Hat and third-party** operators and components (OpenShift, RHACS, RHACM, operators, etc.). **This repository is independent community/open-source material** unless stated otherwise in a given chart. For **production** issues with Red Hat products, use **official Red Hat support** and product documentation—not this repo as your sole escalation path.

## Security

For **security-sensitive** reports, prefer **private disclosure** if GitHub allows (e.g. security advisories for the repository) or contact the maintainer through a channel you already use for sensitive matters; do not post secrets or cluster credentials in public issues.
